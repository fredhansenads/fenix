# SantusERP - Guia do usuario final

Este guia apresenta o uso diario do SantusERP para usuarios administrativos, financeiros, comerciais, operacionais e gestores.

## 1. Acesso

Abra a URL informada pela SANTUS ou pelo administrador da sua empresa.

Credenciais:

- Use seu e-mail corporativo.
- Use a senha temporaria recebida pelo canal seguro definido pela empresa.
- Troque a senha quando solicitado.

Boas praticas:

- Nao compartilhe senha.
- Nao use usuario de outra pessoa.
- Encerre a sessao ao terminar em computadores compartilhados.
- Informe o suporte se notar acesso indevido ou comportamento estranho.

## 2. Dashboard

O dashboard apresenta a visao principal da empresa:

- Entradas e saidas.
- Saldo previsto.
- Clientes ativos.
- Contratos ativos.
- Projetos em andamento.
- Tarefas abertas.
- Alertas de vencimento e pendencias.

Use o dashboard para entender rapidamente a situacao do dia.

## 3. Clientes

Use `Clientes` para consultar e manter a carteira.

Campos principais:

- Nome.
- Tipo.
- Documento.
- E-mail.
- Telefone.
- Status.
- Observacoes.

Rotina recomendada:

1. Criar cliente antes de proposta, contrato ou projeto.
2. Manter e-mail e telefone atualizados.
3. Inativar clientes que nao devem aparecer como ativos.

## 4. Fornecedores

Use `Fornecedores` para controlar prestadores, ferramentas e parceiros.

Rotina recomendada:

- Cadastrar fornecedor antes de criar contas a pagar recorrentes.
- Manter categoria e status atualizados.
- Usar observacoes para dados operacionais nao sensiveis.

## 5. Financeiro

O modulo financeiro possui contas a pagar e contas a receber.

Contas a pagar:

- Descricao.
- Fornecedor.
- Valor.
- Vencimento.
- Status.

Contas a receber:

- Descricao.
- Cliente.
- Valor.
- Vencimento.
- Status.

Boas praticas:

- Atualize status assim que pagar ou receber.
- Revise contas vencidas diariamente.
- Use relatorios para acompanhar periodo.

## 6. Propostas

Use `Propostas` para acompanhar oportunidades comerciais.

Status comuns:

- Rascunho.
- Enviada.
- Aprovada.
- Recusada.
- Expirada.
- Cancelada.

Rotina recomendada:

1. Criar proposta vinculada ao cliente.
2. Definir valor, validade e responsavel.
3. Atualizar status apos retorno do cliente.
4. Acompanhar taxa de conversao nos relatorios.

## 7. Contratos

Use `Contratos` para controlar acordos ativos e vencimentos.

Campos principais:

- Cliente.
- Numero do contrato.
- Titulo.
- Valor.
- Inicio.
- Termino.
- Status.
- Responsavel.

Revise contratos proximos do vencimento para evitar renovacoes perdidas.

## 8. Projetos

Use `Projetos` para organizar entregas por cliente.

Status comuns:

- Planejado.
- Em andamento.
- Pausado.
- Concluido.
- Cancelado.

Boas praticas:

- Defina responsavel e prazo.
- Mantenha status atualizado.
- Vincule tarefas ao projeto sempre que possivel.

## 9. Tarefas

Use `Tarefas` para organizar execucao operacional.

Prioridades:

- Baixa.
- Media.
- Alta.
- Urgente.

Rotina recomendada:

- Revise tarefas abertas no inicio do dia.
- Atualize status durante a execucao.
- Marque como concluida somente quando a entrega estiver finalizada.

## 10. Relatorios

Use `Relatorios` para analisar indicadores e exportar dados.

Possibilidades:

- Resumo executivo.
- Financeiro.
- Comercial.
- Operacional.
- Cadastros.

Quando usar filtro por periodo, confira se as datas selecionadas correspondem ao objetivo da analise.

## 11. Notificacoes

As notificacoes alertam sobre:

- Contas vencidas ou proximas do vencimento.
- Tarefas atrasadas ou proximas do prazo.
- Propostas proximas da validade.
- Contratos proximos do vencimento.

Marque como lida apenas depois de tratar ou reconhecer a pendencia.

## 12. Historico e auditoria

Administradores e gestores podem consultar o historico de atividades.

O historico ajuda a responder:

- Quem criou um registro?
- Quem editou um dado?
- Quando ocorreu uma exclusao?
- Houve tentativa de acao sem permissao?

## 13. Configuracoes

Administradores e gestores podem ajustar:

- Perfil da empresa.
- Usuarios.
- Preferencias.
- Notificacoes.
- Automacoes.
- Saude do sistema.

Evite alterar configuracoes sem entender o impacto para a operacao.

## 14. Como pedir suporte

Ao abrir um chamado, envie:

- Seu nome.
- Empresa.
- Modulo.
- O que estava tentando fazer.
- Mensagem de erro, se houver.
- Print da tela, se possivel.
- Urgencia e impacto no trabalho.

Exemplo:

```text
Modulo: Financeiro
Problema: nao consigo marcar conta como recebida.
Impacto: fechamento mensal parado.
Passos: abri Financeiro > Contas a receber > Editar > Salvar.
Mensagem: permissao negada.
```

## 15. Rotina diaria recomendada

1. Abrir dashboard.
2. Conferir notificacoes.
3. Atualizar tarefas.
4. Atualizar financeiro.
5. Registrar novos clientes, propostas ou projetos.
6. Revisar relatorios quando necessario.

## 16. Cuidados com dados

- Cadastre apenas dados necessarios.
- Evite inserir dados pessoais sensiveis em observacoes.
- Confira destinatarios antes de exportar ou compartilhar relatorios.
- Avise o administrador se notar dados incorretos.
